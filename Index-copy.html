BostonCentral Redesign

In this project, I aimed to analyze and identify flaws in the BostonCentral webpage. I, then, created 3 low-fidelity and 3 high-fidelity prototypes for various screen sizes, and built a responsive website based on those prototypes.

Part I

Picking a web page

BostonCentral

I picked this website because it is very text-heavy and has some room for improvement.

https://www.bostoncentral.com/#

Finding problems

      • The website is very text-heavy and can be overwhelming for some users
      • There are two buttons in different colors for the URL (for example, “Submit an Event” is on the left top and the right top corners)
      • Some groupings of the events seem to be repetitive: it is unclear how “Things to Do” would be different from the “Boston Activities” section as they break down activties in a similar fashion
      • Buttons on the top right corner, including “Submit Event" and “View Newsletter", do not align with the search bar, and some components disappear on a smaller screen
      • In terms of learnability, the web page is fairly straightforward and gives clear explanations of what type of content each button can direct to; however, learnability can be potentially improved by presenting a more diverse range of events groupings to help users find specific events more easily
      • Given the few functionalities the web page needs, it should not be difficult to achieve good memorability; however, as events categories are scattered throughout the web page in a nonmeaningful order, it potentially pose some challenges for users to keep track of all the events

Accessibility

      • the main issue that WebAIM WAVE reported revolved around the lack of alternative texts, and I agree that images without alternative text can pose significant challenges to screen reader users; for example, the icon next to “Submit Event” does not have alternative text, and it would make it very difficult for users to navigate and sign in to their accounts)
      • As I have mentioned in the previous section, WebAIM WAVE also brought up the issue of redundant links, namely adjacent links going to the same URL, and some of them do not include alternative texts either
      • WebAIM WAVE reported a significant amount of usage of layout tables on the website which is not friendly for screen reader users; however, since most of the layout tables only contain two columns – events and dates, it may not be the most prioritized task for developers to fix
